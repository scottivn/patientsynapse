Environment Setup

Get the demo environment running. Takes under 2 minutes.

Prerequisites

  • Python 3.12+
  • Node.js 18+ (for frontend dev server, optional)
  • No EMR credentials needed — stub mode provides all data

Start the Backend

  1. Create/update your .env file in the project root with these values: 
    # .env — Demo configuration
USE_STUB_FHIR=true
APP_ENV=development
ADMIN_DEFAULT_PASSWORD=demo123
LLM_PROVIDER=grok
XAI_API_KEY=your-key-here   # Only needed for fax OCR demo
    💡
    USE_STUB_FHIR=true enables an in-memory FHIR store with 5 patients, insurance coverages, and prescription documents. No EMR connection is required.
  2. Install dependencies 
    pip install -r requirements.txt
  3. Start the server 
    python -m server.main

    Server starts on https://localhost:8443 (or http://localhost:8443 in dev). Look for:

    PatientSynapse starting | env=development | emr=ecw | llm=grok
USE_STUB_FHIR=true — using in-memory FHIR store
Fax inbox: .../IncomingFaxes (150 files pending)
Prescription monitor initialized

Start the Frontend (Optional)

  1. Install and start the Vite dev server 
    cd frontend && npm install && npm run dev

    Frontend runs on http://localhost:5173 and proxies API calls to the backend.

✅
Ready check: Visit http://localhost:8443/api/status. You should see {"status": "running", "emr_provider": "ecw", "llm_provider": "grok", "app_env": "development"}

Workflow 1: Admin Login Admin

JWT-based authentication with HttpOnly cookies and 15-minute inactivity timeout.

  1. Navigate to the login page
    Open http://localhost:5173 (frontend) or http://localhost:8443 (backend with built frontend).
    You'll be redirected to /login automatically.
  2. Enter demo credentials
    FieldValue
    Usernameadmin
    Passworddemo123 (or whatever you set as ADMIN_DEFAULT_PASSWORD)
  3. Login succeeds → redirected to Dashboard
    Admin users land on / (Dashboard). DME users land on /dme/admin.

What to point out

  • JWT stored in HttpOnly cookie — not accessible to JavaScript (XSS-safe)
  • 15-minute inactivity timeout (HIPAA requirement) — auto-logs out and redirects to login
  • Brute-force protection: 5 failed attempts locks the account for 15 minutes
  • HIPAA audit log records every login attempt (success and failure)
🔑
API-only login (for curl/Postman demos):
POST /api/admin/login with body {"username": "admin", "password": "demo123"}
The response sets an access_token cookie.

    Click "Approve". Status changes to Approved.

  4. Send patient confirmation
    Click "Send Confirmation". Choose SMS or Email. A unique token URL is generated.
    Status changes to Patient Contacted.
    POST /api/dme/orders/{id}/send-confirmation
    📱

    Click "Mark Ordered". Enter vendor name and order ID.

    POST /api/dme/orders/{id}/mark-ordered
  7. Mark as shipped
    Enter tracking number and carrier. Status changes to Shipped.
    POST /api/dme/orders/{id}/mark-shipped
  8. Mark as fulfilled
    Click "Fulfill". Order is complete. Status: Fulfilled.
    POST /api/dme/orders/{id}/fulfill

  3. Choose fulfillment method
    Select "Ship to me" or "Pick up in office".
  4. Confirm shipping address (if ship)
    Verify or update the address. Enter:
    FieldValue
    Address9200 Wurzbach Rd

    Order status changes to Patient Confirmed. Staff sees it in the "Patient Confirmed" queue.
    POST /api/dme/confirm/demo-token-sandra-perez

What to point out

  • No login needed — token-based access (similar to password reset links)

  • Minimal PHI exposed — only first name and equipment description visible
  • One-time use — token is invalidated after confirmation

Workflow 4: Fax Ingestion Pipeline Admin

OCR extracts text from incoming faxes, LLM parses clinical data, FHIR client matches to existing patients.

⚠️
Requires LLM API key. The fax pipeline uses your configured LLM provider (Grok/OpenAI/Anthropic) to extract structured data from OCR text. Set XAI_API_KEY (or the relevant key) in .env.
  1. Navigate to Fax Inbox via sidebar → /fax-inbox
    Shows the IncomingFaxes directory with 150+ unprocessed test documents.
  2. Check fax status
    GET /api/faxes/status

    Returns count of pending files vs. already-processed files.

  3. Trigger manual processing
    Click "Process Faxes" or call:
    POST /api/faxes/process

    The pipeline runs: file pickup → OCR (PyPDF2/Tesseract) → LLM extraction → FHIR patient search → referral creation.

  4. Review results
    Successfully processed faxes appear as new referrals in the Referrals tab.
    Each referral shows the LLM-extracted data alongside the original document for verification.

What to point out

  • Supports PDF, TIFF, and JPG fax formats
  • LLM extracts: patient name, DOB, diagnosis codes, referring physician, insurance, and clinical notes
  • FHIR patient matching uses name + DOB fuzzy search
  • Processed files are tracked in DB — won't re-process on next poll
  • The IncomingFaxes/ directory contains 150+ realistic test documents

Scheduling integration

When scheduling a patient with HMO insurance, the system checks for an active referral auth:

Workflow 9: Allowable Rates DME + Admin

Manage insurance reimbursement rates by payer and HCPCS code. Calculate bundle pricing for multi-item orders.

  1. Navigate to Allowable Rates via sidebar → /allowable-rates
  2. Import rates from the bundled spreadsheet
    Upload assets/InsuranceAllowablesForCPAP2026.xlsx via the import feature.
    This contains real-world CPAP/BiPAP insurance reimbursement rates by payer.
    POST /api/allowable-rates/import
  3. Browse rates by payer
    After import, filter by payer name:
    GET /api/allowable-rates?payer=Medicare
  4. Look up a specific rate
    GET /api/allowable-rates/lookup?payer=Medicare&hcpcs_code=E0601&supply_months=6

    Returns the allowed amount for a specific payer + HCPCS + supply period combination.

  5. Use the Bundle Pricing Calculator
    Expand the "Bundle Pricing Calculator" collapsible section at the top of the page.
    1. Select a payer (e.g., Medicare) and supply period (3 or 6 months)
    2. Click a quick preset like "New CPAP Setup" or "Quarterly Resupply (Full Face)" to auto-select the right HCPCS codes
    3. Or manually check individual items (mask, cushion, tubing, filters, etc.)
    4. Click "Calculate" — see per-item rates + total allowed amount

    Items without a rate in the database show "(no rate found)" and are excluded from the total. The "Some rates missing" badge warns when the bundle is incomplete.

    POST /api/allowable-rates/bundle-pricing
  6. Create a custom rate
    POST /api/allowable-rates
    {
  "payer": "Aetna",
  "hcpcs_code": "E0601",
  "description": "CPAP device",
  "supply_months": 6,
  "allowed_amount": 450.00,
  "effective_year": 2026
}

API Explorer (Swagger UI) Dev Only

Full interactive API documentation available in development mode.
